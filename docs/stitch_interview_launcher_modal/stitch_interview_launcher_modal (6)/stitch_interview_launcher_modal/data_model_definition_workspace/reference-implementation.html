Good point. I'd lean towards eventual consistency here since URL shorteners prioritize availability over strict consistency. For collision handling, I'd use a conditional write (like DynamoDB's PutItem with a condition expression) to ensure uniqueness at the storage layer. If the write fails, we retry with a new hash.
